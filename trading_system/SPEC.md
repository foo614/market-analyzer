# ClawdBot Multi-Agent Trading System Specification

## 1. System Overview
The ClawdBot Trading System has been upgraded from a monolithic scheduler script into a decoupled, event-driven Multi-Agent Architecture. This design ensures high fault tolerance, scalability, and clean separation of concerns, allowing discrete components (Agents) to operate asynchronously.

## 2. Architecture: The Message Bus
At the core of the system is a lightweight, file-based message queue (`message_bus.py`) located in `trading_system/agents/bus_data/`. 
This allows cross-language communication (Python & Node.js) without the overhead of Redis or RabbitMQ.

### Active Queues (Topics):
- `market_data`: Raw data payloads fetched by sensors.
- `trade_signals`: Actionable `BUY/SELL` instructions generated by the Quant brain.
- `notifications`: Formatted strings or objects meant for human consumption.
- `system_state`: Heartbeat and health statuses of active agents.

## 3. Agent Specifications

### 3.1 Data Agent (`data_agent.py` & `obv_monitor.js`)
**Role:** The Sensory System.
**Responsibilities:**
- Continuously poll external APIs (Yahoo Finance, Alpha Vantage) during market hours.
- Calculate high-frequency metrics:
  - Intraday Volume Profile (Surge/Shrink detection).
  - Price-Volume Divergence (VWAP).
  - OBV (On-Balance Volume) and RSI.
- **Output:** Pushes structured JSON market anomalies to the `market_data` queue.

### 3.2 Quant Agent (`quant_agent.py`)
**Role:** The Strategy Brain.
**Responsibilities:**
- Consumes data from the `market_data` queue or fetches historical data for daily bars.
- Applies optimized technical logic (e.g., `RSI < 30` + `OBV 7/10 Crossover`).
- Runs daily parameter grid-search optimizations (Backtest Framework) to self-adjust thresholds.
- **Output:** Emits explicit JSON trade objects (e.g., `{"symbol": "TSLA", "action": "BUY", "amount": 500}`) to the `trade_signals` queue.

### 3.3 Execution Agent (`execution_agent.py`)
**Role:** The Hands (Order Gateway).
**Responsibilities:**
- Strictly listens to the `trade_signals` queue.
- Translates system signals into eToro API payload format.
- Executes trades on the eToro Virtual/Demo Portfolio via REST API.
- Implements basic sanity checks (e.g., validating amount limits).
- **Output:** Pushes trade execution results (Success/Fail) to the `notifications` queue.

### 3.4 Notification Agent (`notification_agent.py`)
**Role:** The Mouthpiece.
**Responsibilities:**
- Strictly listens to the `notifications` queue.
- Formats system events, trade results, and LLM market briefs into human-readable Markdown.
- Interfaces with the Telegram Bot API to broadcast messages to the user.
- Manages deduplication (ensuring the user is not spammed with identical alerts).

## 4. Sub-Modules & Utilities

### Market Analyzer (`market_analyzer.py`)
- Runs pre-market (8:30 AM EST) and post-market.
- Analyzes VIX, S&P 500, NASDAQ, and Futures premiums.
- Passes raw data to OpenAI (`gpt-4o-mini`) to generate a concise, Chinese-language trader brief.

### eToro Tracker (`etoro_tracker.py`)
- Runs post-market (4:30 PM EST).
- Fetches closed trades and portfolio status from the eToro API.
- Calculates Win Rate, Net PnL, P/L Ratio, and Max Drawdown.
- Saves to a local SQLite Database (`etoro_trades.db`).

---

## 5. Deployment & Process Management

- **Master Launcher:** `start_all_agents.py`
  - A Python script that spawns the 4 core agents as independent background processes using `subprocess.Popen`.
- **ClawdBot Integration:**
  - The master ClawdBot reads `HEARTBEAT.md` to ensure `start_all_agents.py` is running. If the processes die, ClawdBot will automatically revive the entire ecosystem.

## 6. Future Roadmap
1. **Reinforcement Learning (RL):** Replace the static `Quant Agent` with a dynamic PPO/SAC model that learns from the `etoro_trades.db` outcomes.
2. **Hard Stop-Loss Module:** Implement a global circuit breaker in the `Execution Agent` to halt trading if daily portfolio drawdown exceeds 1%.
3. **Sentiment Agent:** A new agent dedicated to scraping X (Twitter) and Reddit for stock-specific hype spikes.
